## Loop Constraints

### Read-Only Constraint
- ClawDBot only generates review requests
- ClawDBot does not perform reviews
- ClawDBot does not make review decisions
- ClawDBot does not implement review outcomes

### Deterministic Constraint
- Same inputs produce same outputs
- No randomness in review request generation
- Reproducible review request generation

### Explainable Constraint
- All review requests include explanations
- All review priorities are documented
- All review urgencies are documented

## Human Review Process

### Review Steps

1. **Receive Review Requests**
   - Human receives review requests from ClawDBot
   - Reviews request descriptions and explanations
   - Assesses review priority and urgency

2. **Perform Reviews**
   - Reviews ClawDBot outputs
   - Evaluates output quality and accuracy
   - Makes review decisions (approve, reject, revise)

3. **Provide Feedback**
   - Provides feedback on reviews
   - Updates review status
   - Documents review outcomes

4. **Implement Outcomes** (if approved)
   - Implements approved outcomes
   - Monitors implementation results
   - Updates review process based on results
