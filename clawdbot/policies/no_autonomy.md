# No Autonomy Policy

**Generated**: January 25, 2026  
**Purpose**: Explicit policy defining ClawDBot's non-autonomous nature (NO CODE)

---

## Policy Statement

ClawDBot is **strictly non-autonomous**. It never runs on its own, never makes decisions autonomously, and never acts without explicit human invocation or approval.

---

## Core Principles

### 1. Always Invoked

#### Definition
ClawDBot is always invoked by an external entity (runner, CLI, human). It never self-activates.

#### Application
- ClawDBot CLI requires manual execution
- ClawDBot Runner requires manual start
- No scheduled execution without explicit setup
- No autonomous triggers

#### Violations
- Autonomous execution
- Self-triggering
- Scheduled execution without explicit setup

---

### 2. No Autonomous Decisions

#### Definition
ClawDBot never makes decisions autonomously. All decisions require human approval.

#### Application
- All outputs require human review
- All recommendations require human approval
- All drafts require human approval
- No autonomous action on insights

#### Violations
- Autonomous decision-making
- Auto-approval without human review
- Autonomous action on insights

---

### 3. No Autonomous Actions

#### Definition
ClawDBot never performs actions autonomously. All actions require explicit human approval.

#### Application
- No autonomous publishing
- No autonomous fixes
- No autonomous updates
- No autonomous changes

#### Violations
- Autonomous publishing
- Autonomous fixes
- Autonomous updates
- Autonomous changes

---

## Policy Enforcement

### Enforcement Mechanisms

1. **Read-Only Design**
   - ClawDBot has no side effects
   - ClawDBot cannot modify data
   - ClawDBot cannot trigger actions

2. **Human Gates**
   - All outputs require human review
   - All recommendations require human approval
   - All drafts require human approval

3. **Manual Invocation**
   - ClawDBot CLI requires manual execution
   - ClawDBot Runner requires manual start
   - No autonomous scheduling

---

## Policy Violations

### Violation Types

1. **Autonomous Execution**
   - ClawDBot runs without explicit invocation
   - ClawDBot self-triggers
   - ClawDBot schedules itself

2. **Autonomous Decisions**
   - ClawDBot makes decisions without human approval
   - ClawDBot auto-approves outputs
   - ClawDBot acts on insights autonomously

3. **Autonomous Actions**
   - ClawDBot publishes autonomously
   - ClawDBot fixes autonomously
   - ClawDBot updates autonomously

### Violation Consequences

1. **Immediate Stop**: Stop violating process immediately
2. **Log Violation**: Log all violations for audit
3. **Alert Admin**: Alert administrators of violations
4. **Review Process**: Review process to prevent future violations

This policy has **no exceptions**. ClawDBot is always non-autonomous, regardless of circumstances.
