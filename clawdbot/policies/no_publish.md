# No Publish Policy

**Generated**: January 25, 2026  
**Purpose**: Explicit policy prohibiting ClawDBot from publishing content autonomously (NO CODE)

---

## Policy Statement

ClawDBot **never publishes content autonomously**. It generates drafts only, and all publishing requires explicit human approval.

---

## Core Principles

### 1. Draft Generation Only

#### Definition
ClawDBot only generates drafts. It never publishes content.

#### Application
- Generate editorial drafts
- Generate social media drafts
- Generate review drafts
- No publishing

#### Violations
- Publishing content
- Auto-publishing
- Publishing without approval

---

### 2. Human Approval Required

#### Definition
All publishing requires explicit human approval.

#### Application
- All drafts require human review
- All publishing requires human approval
- No autonomous publishing
- No auto-publishing

#### Violations
- Publishing without approval
- Auto-publishing
- Autonomous publishing

---

### 3. Runner Responsibility

#### Definition
Publishing is the responsibility of the runner, not ClawDBot.

#### Application
- Runner handles publishing
- Runner checks approval requirements
- Runner sends messages (with approval)
- ClawDBot only generates drafts

#### Violations
- ClawDBot publishing directly
- ClawDBot sending messages
- ClawDBot bypassing runner

---

## Policy Enforcement

### Enforcement Mechanisms

   - ClawDBot cannot publish
   - ClawDBot cannot send messages

   - Runner checks approval requirements
   - No publishing without approval

3. **Runner Responsibility**
   - Runner handles all publishing

   - ClawDBot only generates drafts
