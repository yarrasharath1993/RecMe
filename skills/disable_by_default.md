# Disable by Default Policy

**Generated**: January 25, 2026  
**Purpose**: Policy requiring all skills to be disabled by default

---

## Policy Statement

All new skills are **disabled by default**. Skills must be explicitly enabled after review and approval.

---

## Core Principles

### 1. Disable by Default

#### Definition
All new skills start in disabled state.

#### Application
- New skills are disabled when added to registry
- Skills remain disabled until explicitly enabled
- Skills require explicit enablement

#### Rationale
- Prevents unintended side effects
- Ensures skills are reviewed before use
- Maintains control over skill activation

---

### 2. Explicit Enablement

#### Definition
Skills must be explicitly enabled after review.

#### Application
- Skills require explicit enablement
- Enablement requires review
- Enablement requires approval

#### Rationale
- Ensures skills are reviewed before use
- Maintains control over skill activation
- Prevents accidental activation

---

### 3. Review Before Enablement

#### Definition
Skills must be reviewed before enablement.

#### Application
- Review skill against admission checklist
- Verify skill meets criteria
- Approve skill for enablement

#### Rationale
- Ensures skills meet quality standards
- Prevents problematic skills from being enabled
- Maintains skill quality

---

## Enablement Criteria

### Criteria for Enablement

1. **Admission Approved**: Skill has been approved for admission
2. **Tests Passed**: Skill tests have passed (if applicable)
3. **Documentation Complete**: Skill documentation is complete
4. **Rollback Tested**: Skill rollback has been tested
5. **Side Effects Approved**: Side effects have been approved (if applicable)

---

## Enablement Process

### Step 1: Review Skill

1. Review skill against admission checklist
2. Verify skill meets criteria
3. Test skill functionality (if applicable)
4. Test skill rollback (if applicable)

### Step 2: Approve Enablement

1. Approve skill for enablement
2. Document approval decision
3. Set enablement conditions (if any)

### Step 3: Enable Skill

1. Enable skill in registry
2. Update skill status
3. Document enablement
4. Monitor skill usage

---

## Disablement Process

### Step 1: Identify Need for Disablement

1. Identify issues with skill
2. Identify violations of policies
3. Identify safety concerns
4. Identify performance issues

### Step 2: Disable Skill

1. Disable skill in registry
2. Update skill status
3. Document disablement reason
4. Notify users (if applicable)

### Step 3: Review Disablement

1. Review disablement decision
2. Determine if skill can be fixed
3. Determine if skill should be deprecated
4. Determine if skill should be removed

## Exceptions

### No Exceptions

This policy has **no exceptions**. All skills are disabled by default, regardless of circumstances.
